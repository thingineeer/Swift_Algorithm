 ```
로컬 환경에서 작업 후 Add -> Commit -> Push -> Pull Request의 과정을 거친다.
   
   Prefix
   > [Feat] : 새로운 문제 업로드
   > [Setting] : 파일 정리
   > [Fix] : 버그, 오류 해결, 문제의 테스트 케이스가 바뀌어 정답이 달라질때
   > [Refactor] : 시간 복잡도, 공간복잡도 줄이고 코드 전면 개선할때 사용
   > [Style] : 코드 포맷팅, 코드 변경이 없는 경우, 주석 수정
   > [Docs] : README 수정할때 사용

```
